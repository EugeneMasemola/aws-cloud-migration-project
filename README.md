# AWS Website Migration Case Study

## Overview

This project is a sanitized case study based on website migration and AWS provisioning work completed during my time at Didusec.

The work involved supporting the migration of websites from traditional hosting environments to AWS, including planning, provisioning, DNS preparation, website transfer support, validation, and documentation.

> Note: This repository does not contain proprietary company code, client data, credentials, internal documents, or confidential infrastructure details.

## Projects Represented

- Migration of Didusec website from xneelo to AWS
- Migration and AWS provisioning for Lifa Insurance Brokers website

## Objectives

The main objectives of the migration work were to:

- Move website hosting from traditional hosting to AWS
- Improve cloud readiness
- Prepare scalable hosting infrastructure
- Support DNS and domain transition activities
- Validate website availability after migration
- Document the migration process and lessons learned

## My Role

I contributed to the project by supporting:

- AWS infrastructure preparation
- Website migration planning
- Hosting environment setup
- DNS and deployment validation
- Migration documentation
- Troubleshooting during and after migration

## Technologies and Concepts

- Amazon Web Services

- Technical documentation

## High-Level Migration Process

1. Reviewed the existing hosting setup.
2. Prepared the AWS hosting environment.
3. Backed up website files and related assets.
4. Supported the transfer of website content.
5. Assisted with DNS preparation and cutover.
6. Validated that the website was accessible after migration.
7. Documented issues, fixes, and lessons learned.
